# Project Memory

## Tech Stack

The tech stack includes:
- **Frontend Framework**: React.js
- **Build Tool**: CRACO (Custom React Scripts)
- **CSS Preprocessor**: PostCSS
- **Code Editor**: CodeMirror
- **Third-Party Plugins**: Webpack Health Plugin, Dev Server Setup, Babel Metadata Plugin
- **Core Libraries**: ReactDOM, React Router DOM

## Core Architecture

The application architecture is modular, structured into several components:

1. **Chat Components**:
	- `ChatInput.jsx`, `ChatPanel.jsx`, `MessageItem.jsx`, `MessageList.jsx`
2. **Consensus Components**:
	- `ConsensusRoom.jsx`
3. **Editor Components**:
	- `CodeEditor.jsx`, `EditorPanel.jsx`, `FileTree.jsx`
4. **Layout Components**:
	- `Header.jsx`, `MainLayout.jsx`, `MobileLayout.jsx`, `ResponsiveLayout.jsx`
5. **Preview Components**:
	- `PreviewPanel.jsx`
6. **Progress Components**:
	- `ProgressPanel.jsx`
7. **Project Dialog**:
	- `ProjectDialog.jsx`
8. **Settings Components**:
	- `ApiKeySettings.jsx`

## Current Status (Initialized)

- **Repository Setup**: The repository is initialized and structured for a React.js project.
- **Plugins and Configurations**: Custom plugins are in place for health checks and visual edits.
- **Initial Components**: The `src/components` directory includes a variety of React components for chat, consensus, editor, layout, preview, and settings.

## Deployment Targets

- **Deployment Platform**: Vercel
- **Build Configuration**: CRACO is configured for custom build processes.
- **Custom Plugins**: Webpack Health Plugin and Babel Metadata Plugin are integrated into the build pipeline.
